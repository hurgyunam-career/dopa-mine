# UI Design

제품의 시각적 일관성과 구현 기준을 정의합니다.

## 1) 플랫폼 및 레이아웃
- 기본 전략: Mobile First (iOS/Android)
- 데스크톱 디버깅 기준 창 크기: `400x800`
- 반응형 정책: 콘텐츠 최대 폭 `600px`, 넓은 화면에서는 중앙 정렬 + 배경 확장

## 2) 디자인 시스템
- 테마: Material 3
- 모드: Dark Mode 우선
- 기본 색상: Deep Purple
- 배경색: `Color(0xFF121212)`
- 타이포그래피: 가독성 높은 Sans-serif 계열 (예: Pretendard)

## 3) 핵심 UX 원칙
- 최소 인지 부하: 불필요한 장식/애니메이션 최소화
- One Action Per Screen: 화면당 핵심 액션 1개를 명확히 강조
- Thumb Zone 우선: 시작/완료 같은 주요 버튼은 화면 하단 영역 배치

## 4) 컴포넌트 기준
- 카드: `BorderRadius.circular(16)`, 선택 상태는 `BorderSide`로 구분
- 버튼: 하단 고정형(Sticky Bottom), 최소 높이 `60px`
- 간격: 화면 기본 패딩/여백 단위를 일관되게 유지

## 5) 구현 매핑 체크리스트
- 홈 화면: 카드 선택 상태가 명확히 보이는가
- 세션 화면: 주요 액션 버튼이 하단 고정되어 있는가
- 리포트 화면: 핵심 결과(시간/횟수/포인트)가 1스크린 내 우선 노출되는가
- 공통: 콘텐츠 폭 제한(`<= 600px`)이 적용되는가
